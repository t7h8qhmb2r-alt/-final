# BAR_LAB.EXE アプリ化へのロードマップ

現在のデザイン（Next.js + Tailwind CSS）を活かしてアプリ化するための、推奨される技術スタックとステップを提案します。

## 1. 推奨される開発フレームワーク

現在のHTML/CSS構成と最も相性が良く、効率的に「アプリ」として公開できる方法は以下の2つです。

### A. PWA (Progressive Web App) として公開
**もっとも手軽な方法です。**
*   **特徴**: Webサイトをスマホのホーム画面にアイコンとして追加し、アプリのように振る舞わせます。
*   **メリット**: ストア審査が不要で、現在のNext.jsの構成をほぼそのまま使えます。
*   **向いているケース**: 素早くユーザーに届けたい、プッシュ通知を段階的に導入したい場合。

### B. Capacitor (Ionic) を使ったハイブリッドアプリ化
**ストア（App Store / Google Play）での公開を目指す場合。**
*   **特徴**: 現在のWebコードを「ネイティブアプリの殻」で包み込みます。
*   **メリット**: カメラ、位置情報、通知などのスマホ機能をフルに活用でき、ストアからダウンロード可能になります。
*   **向いているケース**: 「本格的なアプリ」として信頼性を高めたい、ストアからの集客を狙いたい場合。

## 2. 技術スタック詳細 (Recommended)

| カテゴリ | 推奨技術 | 理由 |
| :--- | :--- | :--- |
| **Frontend** | **Next.js (App Router)** | 現在のコード資産をそのまま活用可能。 |
| **Styling** | **Tailwind CSS** | 現在のデザインシステムを完全に再現。 |
| **Backend** | **Supabase** | 認証（ログイン）やデータベース（お店データ、予約）を爆速で構築可能。 |
| **Maps** | **Mapbox / Google Maps API** | 「Map View」画面の実装に不可欠。 |
| **Deployment** | **Vercel** | Web/PWAとしての公開に最適。 |

---

## 3. 次のステップ（実装へのガイド）

1.  **コードの書き出し**: 各画面の「View Code」から、コンポーネントごとのコードを取得します。
2.  **ロジックの実装**: 
    *   `route.ts` にあるようなAI検索ロジックの強化。
    *   予約フォーム（Booking Entry）からデータベースへの保存処理。
3.  **地図の連動**: Mapboxなどのライブラリを使い、保存した「検体（店舗）」の座標を表示。
